# Trust Language Rules

Guidelines for trust, verification, and identity language on the ECZ-ID Developer Gateway.

## Core Language Principles

### 1. Verified, Not Claimed

**Do:** "Verified identity" / "Attestable provenance" / "Verifiable through Resolver"
**Don't:** "Trusted" (by itself) / "Guaranteed" / "Certified" (unless actually certified)

Identity is verified, not assumed. Every claim on this site should be about verifiability, not about trust-by-assertion.

### 2. Infrastructure, Not Product

**Do:** "Trust infrastructure" / "Identity layer" / "Verification model"
**Don't:** "Trust product" / "Identity app" / "Verification tool"

ECZ-ID is infrastructure. It's foundational, not transactional. Language should reflect permanence and durability.

### 3. Enabled, Not Controlled

**Do:** "ECZ-ID enables verified identity" / "Organisations deploy ECZ-ID"
**Don't:** "ECZ-ID controls identity" / "ECZ-ID owns your identity"

ECZ-ID is a layer that enables sovereignty. It does not imply ownership or control.

### 4. Precise, Not Hyperbolic

**Do:** "Covers nine identity domains" / "Supports all four drone classes"
**Don't:** "Revolutionary" / "Game-changing" / "The world's best"

Let the breadth and capability speak. Precision is more persuasive than superlatives.

## Tone Rules

### Use

| Quality | Description |
|---------|-------------|
| Safe | The reader feels confident and assured |
| Calm | No urgency, no pressure, no anxiety |
| Precise | Specific, measurable, technical |
| Premium | High-quality, polished, professional |
| Corporate | Suitable for enterprise and government audiences |
| Infrastructure-grade | Foundational, durable, serious |
| Durable | Language that won't age poorly |
| Inevitable | ECZ-ID as the logical, necessary infrastructure |
| Trustworthy | Earned through accuracy, not claimed |

### Avoid

| Quality | Description |
|---------|-------------|
| Neon / Cyberpunk | No tech-noir aesthetics or hacker culture references |
| Hackerish | No terminal aesthetics or "1337" language |
| Chaotic | No information overload or scattered messaging |
| Gimmicky | No tricks, no viral tactics, no memes |
| Speculative | No "imagine if" or "what if" futures |
| Hypey | No "revolutionary", "game-changing", "disruptive" |
| Meme-like | No internet humour or trendy references |
| Startup-campy | No "move fast and break things" energy |

## Specific Language Guidelines

### Referring to ECZ-ID

- "ECZ-ID" — always capitalised, always hyphenated
- "ECZ-ID trust infrastructure" — not "ECZ-ID platform" or "ECZ-ID service"
- "ECZ-ID identity" — when referring to a specific entity's identity
- "An ECZ-ID" — when referring to a specific identity instance

### Referring to EcoCitizenz

- "EcoCitizenz" — exact capitalisation
- "An EcoCitizenz infrastructure product" — when describing ECZ-ID's provenance
- Never: "EcoCitizenZ", "Eco Citizenz", "eco citizenz"

### Referring to TrustOps

- "TrustOps" — exact capitalisation
- "Provision through TrustOps" — for any sales, pricing, or credential action
- "All provisioning happens through TrustOps" — when redirecting from this site
- Link to: `https://trustops.ecocitizenz.com/start`

### Referring to Resolver

- "Resolver" — capitalised
- "Verify through Resolver" — for any verification or proof action
- "Resolver returns..." — when describing verification results
- Link to: `https://resolver.ecocitizenz.org`

### Verification Language

- "Verified" — identity has been through the verification process
- "Attestable" — claims that can be verified by others
- "Attested" — claims that have been verified and recorded
- "Trust score" — always lowercase "score"
- "Provenance" — the traceable history of an entity's actions
- "Identity-anchored" — linked to a verified identity

### Words to Use

- Verified, attestable, provenance, identity-anchored
- Infrastructure, layer, foundation, backbone
- Provision, deploy, integrate, configure
- Continuous, real-time, runtime
- Audit trail, compliance-ready, regulatory-aligned
- Domain-complete, standards-aligned, infrastructure-grade

### Words to Avoid

- Revolutionary, disruptive, game-changing
- Guaranteed, bulletproof, unhackable
- Best, fastest, cheapest (comparative claims)
- Trust (as a standalone adjective without "verified" or "attestable")
- Blockchain, Web3, token (unless specifically relevant)
- AI-powered (when describing ECZ-ID itself)
- Seamless, frictionless (overused, vague)

- "Explore All Domains" — for domain discovery

## Examples

### Good

> "ECZ-ID provides verified identity for every AI agent, enabling trusted interactions, auditable decisions, and provenance-tracked outputs across the machine economy."

> "Provision your agent's identity through TrustOps. Verify it through Resolver. Every interaction becomes attestable."

> "Sign up now for our game-changing identity solution before it's too late!"
